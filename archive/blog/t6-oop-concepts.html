Week 6 - Blocks, procs and lambdas

2015/7/5

Here is a brief, basic introduction to the concept of closures in Ruby. Closures are simply a programming concept where you create a method in such a way that when you can call it from a different part of your program, it retains its original scope. For example, it still has access to the variables in its original environment. Some of the ways Ruby provides support for closures are blocks, procs, and lambdas.

A block is a chunk of code between {}brackets. The {}bracketed part of a #each or #map call, as in array.each { |item| puts item }, is an example of a block. Blocks can be called from elsewhere in the program, typically with the yield method. Blocks can only be used as part of a call to a method, unlike a proc.

A proc (short for "procedure") is essentially a block wrapped up in an object. A proc object is an instance of the Proc class. Because it is an object, it has a name and can be passed around like any other variable. It can also receive messages.

A lambda is a specialized proc; a lambda object is even an instance of the Proc class. There are a number of differences that generally make lambdas more restrictive than procs. For example, procs don't care how many arguments you send it, while lambdas complain if you send the wrong number. Also, the return keyword is treated differently–in a lambda, return sends the scope right outside of the brackets, while a proc exits the entire method on return.
