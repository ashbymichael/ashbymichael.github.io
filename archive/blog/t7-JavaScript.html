Week 7 - Looping in Javascript

2015/7/12

Ruby makes looping through collections (i.e. hashes and arrays) simple with its myriad built-in methods, but Javascript makes you work a little harder. Let's look at how they differ.

While Ruby offers at least half a dozen different options to cycle through various kinds of collections–and several variations on each–Javascript offers just two keywords for looping: while and for. while is the more basic of the two, accepting only one argument: a stop condition. For example, the expression while ( x <= 10 ) { ... } would execute the code inside the braces, check the value of x and, if it is less than or equal to 10, execute the code block again. If the code block never causes x to be greater than 10, your program will enter an infinite loop.

The for keyword offers essentially the same functionality as while, but it makes a couple of common tasks easier. First is easy to create an infinite loop when using while, and the primary purpose of for is to make it easier to create an iterator. The code for (i = 0; i < 10; i++ ) { ... } will execute the code in the block continuously until it reaches 10, but unlike a simple while loop, you have considerable control over when it stops. The other use of for is as an enumerator function similar to Ruby's .each function. It uses the following syntax: for ( n in numbers ) { ... }.
