Week 5 - Ruby Classes

2015/6/22

In Ruby, you can use classes to model real-world things and concepts. For example, you may want to model users when building a web app, so that they can create accounts, log in/out, etc... A typical Users class might look something like this:

Class Users
 def initialize(username, email, password)
  @username = username
  @email = email
  @password = password
 end
 attr_accessor :username, :email, :password
end

The code above creates a Users class and gives it three attributes, a username, an email, and a password. (In an actual web application, this is not good practice for storing passwords. It's fine for explanatory purposes, though.) It also uses attribute accessors to allow you to read and change the attributes. The variables that begin with an "@" are instance variables, meaning that they are unique to each instance of Users.

From this Users class, we can create as many instances as we want. Each instance represents a user, and has its own @username, @email and @password values. Thanks to attr_accessor, we can check these values at any time using dot notation, for example: user_1.username #=> "Muttons". We can assign the attributes new values (seemingly) just like any other value, for example: user_1.username = "Mittens".
